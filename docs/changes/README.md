# docs/changes

이 폴더는 `지옥` 프로젝트의 변경 기록을 모아두는 곳입니다.

## handoff 문서와 change 문서의 차이

| 구분 | handoff | change |
|------|---------|--------|
| 목적 | 다음 담당자가 바로 이어서 작업할 수 있도록 현재 상태를 인계 | 무엇을 왜 바꿨는지 변경 이력을 기록 |
| 기준 시점 | 특정 작업 완료 시점 | 변경이 발생할 때마다 |
| 내용 | 현재 상태, 결정사항, 다음 작업 | 변경 전/후, 이유, 영향 범위 |
| 작성자 | 작업을 완료한 담당자 | 변경을 반영한 담당자 |

## 변경 기록 파일 네이밍 규칙

`YYYY-MM-DD-주제.md`

예시:
- `2026-04-14-main-routing-update.md`
- `2026-04-15-security-policy-change.md`
- `2026-04-16-response-breakpoint-fix.md`

## change 문서를 남겨야 하는 경우

- 구조 변경 (라우팅, 폴더, 컴포넌트 경계)
- 정책 변경 (보안, 권한, 열람 제한)
- UI 방향 수정 (운영형/소개형 판단 기준 변경)
- 반응형 기준 수정 (브레이크포인트, 토큰 변경)
- 보안/권한 규칙 수정
- API 계약 변경
- 기존 결정사항 취소 또는 변경

## change 문서를 별도로 남기지 않아도 되는 경우

- 단순 오타 수정
- 의미 없는 미세 변경
- 문서 서식/주석 정리
- 이미 change 문서로 기록된 변경의 자연스러운 후속 수정
