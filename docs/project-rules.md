# Hell Project Common Rules

## 1. 목적

이 문서는 `지옥` 프로젝트의 공통 작업 기준을 정의합니다.

- React 기반 반응형 웹사이트를 일관된 구조로 운영합니다.
- 불필요한 기능 확장을 막고, 재사용성과 유지보수성을 우선합니다.

## 2. 기본 원칙

1. 추측하지 않습니다. 근거가 없으면 `가정`으로 표시합니다.
2. 기존 결정사항을 임의로 뒤집지 않습니다.
3. 모든 제안에는 짧게라도 이유를 함께 적습니다.
4. 모바일 우선을 포함한 반응형 대응을 기본 전제로 둡니다.
5. 컴포넌트는 재사용성과 책임 분리를 우선합니다.
6. 전역 상태관리는 실제 필요가 확인되기 전까지 도입하지 않습니다.
7. 페이지 완성보다 구조 안정화가 우선인 단계에서는 최소 구현만 합니다.

## 3. 코드 구조 원칙

1. `pages`는 라우트 단위 화면만 담당합니다.
2. `components`는 재사용 가능한 UI 조각을 담당합니다.
3. `layouts`는 공통 골격과 배치를 담당합니다.
4. `hooks`는 UI 로직 재사용이 명확할 때만 추가합니다.
5. `utils`는 순수 함수, 상수, 라우트 정의 등을 둡니다.
6. `styles`는 토큰, 리셋, 글로벌, 유틸리티 순으로 관리합니다.

## 4. 반응형 원칙

1. 기본 스타일은 모바일 기준으로 작성합니다.
2. 브레이크포인트는 토큰에 정의하고 페이지별로 중복 선언하지 않습니다.
3. 레이아웃은 고정 폭보다 `container`, `grid`, `clamp`를 우선 사용합니다.
4. 텍스트와 간격은 화면 크기 변화에 따라 무너지지 않도록 설계합니다.

## 5. 변경 원칙

1. 수정 시 무엇을 왜 바꾸는지 명확히 남깁니다.
2. 불필요한 파일 추가나 추상화는 지양합니다.
3. 후속 에이전트가 이어받을 수 있도록 변경 범위를 문서에 기록합니다.
