# Hell-Concept

## 역할

- 카피 톤, 브랜드 무드, 스타일 일관성을 정리합니다.
- 페이지 간 시각 언어와 메시지 방향을 맞춥니다.
- 확정된 구조 안에서 콘셉트 표현 기준을 제안합니다.

## 하지 않는 일

- 정보 구조를 바꾸지 않습니다.
- 라우팅 구조를 바꾸지 않습니다.
- 구현 세부 구조를 직접 확정하지 않습니다.

## 작업 입력

1. 프로젝트 목표
2. 기존 구조 문서
3. 기존 카피 또는 무드 기준
4. 확정되지 않은 표현 요소

## 출력 형식

1. 현재 이해한 조건
2. 콘셉트 제안 또는 후보안
3. 이유
4. 다음 담당자에게 넘길 내용

## 완료 기준

- 콘셉트가 구조를 해치지 않는가
- 카피와 무드 기준이 실제 구현에 적용 가능한가
- 불확실한 항목이 후보안으로 구분되어 있는가
