# Agents

이 폴더는 `지옥` 프로젝트의 역할별 에이전트 문서와 공통 인계 템플릿을 함께 관리합니다.

이 문서들을 분리해 두는 목적은 다음과 같습니다.

- 각 에이전트의 책임 범위를 고정합니다.
- 역할 충돌을 줄입니다.
- Hell-Orchestrator가 현재 요청에 맞는 담당자를 선택할 수 있는 기준 문서를 제공합니다.
- 작업 인계 형식을 통일합니다.

## 포함 구조

### 관리

- `hell-orchestrator.md`
  - 사용자 요청을 먼저 해석하고, 현재 로그·handoff·기존 결정사항을 기준으로 가장 적절한 담당자를 선택하고 작업 흐름을 통제합니다.

### 기획/설계

- `hell-planner.md`
  - 프로젝트 목표, 사용자 흐름, 페이지 우선순위, 확정사항과 가정사항을 정리합니다.
- `hell-architect.md`
  - 정보 구조, 라우팅 구조, 레이아웃 구조, 페이지 책임, 컴포넌트 경계를 설계합니다.

### 콘텐츠/데이터

- `hell-ux-writer.md`
  - 운영형 화면에 맞는 카드 제목, 상태 라벨, 메타 정보, 버튼 라벨, 안내 문구를 정리합니다.
- `hell-data-modeler.md`
  - 더미 데이터와 운영 객체 구성을 실제 서비스처럼 보이도록 재구성합니다.

### 비주얼/디자인

- `hell-visual-designer.md`
  - 로고, 아이콘, 배지, 태그, 상태칩, 그래픽 모티프, 섹션별 시각 리듬, MVP 인상 완화를 담당합니다.
  - 구조를 바꾸지 않고 시각적 강약과 스타일 차이를 설계합니다.

### 구현/구조 감시

- `hell-builder.md`
  - React 컴포넌트, 레이아웃, 라우팅 연결 등 실제 화면과 기능을 구현합니다.
- `hell-system-keeper.md`
  - 폴더 책임, 토큰 관리, 구조 중복, 과도한 추상화 여부를 점검합니다.

### 검수

- `hell-responsive-auditor.md`
  - 모바일·태블릿·데스크톱 기준의 반응형 품질, 오버플로우, 카드 위계, 브레이크포인트 사용 여부를 검수합니다.
- `hell-qa.md`
  - UX 흐름, 접근성, 구조 준수, 실패 컷 규칙 통과 여부를 최종 검수합니다.

### 보안/서버

- `hell-security.md`
  - 인증, 권한, 열람 제한, 공개 범위, 마스킹, 열람 이력, 차단 정책을 정리합니다.
- `hell-backend.md`
  - API, 데이터 모델, 인증 흐름, 상태 전이, 서버 책임 범위를 설계합니다.

### 공통 템플릿

- `handoff-template.md`
  - 여러 에이전트가 같은 프로젝트를 이어서 작업할 때 사용하는 공통 인계 템플릿입니다.
  - 개별 역할 문서가 아니라, 다음 담당자가 바로 이어받을 수 있도록 현재 상태·변경 파일·유지 결정·가정·다음 작업을 남기는 공통 기준 문서입니다.

## 운영 원칙

- 각 에이전트는 자신의 역할 문서 범위 안에서만 판단합니다.
- 역할 밖 판단이 필요하면 해당 역할 담당자에게 다시 넘깁니다.
- Hell-Orchestrator는 이 폴더의 문서들을 기준으로 현재 요청에 맞는 담당자를 선택합니다.
- handoff-template.md는 모든 역할이 공통으로 참고하는 인계 기준 문서로 유지합니다.

## 역할 경계 구분

| 영역 | 담당자 |
|------|--------|
| 문구/라벨/운영형 인터페이스 언어 | Hell-UX Writer |
| 데이터 문맥/더미 데이터/상태값 | Hell-Data Modeler |
| 시각 자산/로고/아이콘/배지/시각 리듬/MVP 인상 완화 | Hell-Visual Designer |
| 정보 구조/라우팅/레이아웃 | Hell-Architect |
| React 구현 | Hell-Builder |
