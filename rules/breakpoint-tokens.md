# Breakpoint Tokens

## 목적

- 반응형 기준값을 공통 토큰으로 관리합니다.
- Hell-System Keeper와 Hell-Responsive Auditor가 브레이크포인트 관련 문제를 판단하는 기준을 제공합니다.

## 권장 위치

- `src/styles/tokens/breakpoints.ts`
- 또는 `src/styles/tokens.css`

## 기본 기준

| 기준 | 최소 너비 |
|------|-----------|
| mobile | 기본값 |
| tablet | `768px` 이상 |
| desktop | `1200px` 이상 |

## 강제 규칙

1. 브레이크포인트 값은 한 곳에서만 관리합니다.
2. 컴포넌트 내부에 임의 숫자를 반복 선언하지 않습니다.
3. 레이아웃 전환이 필요한 지점만 조건을 추가합니다.
4. 단순 크기 확대보다 구조 유지 여부를 먼저 확인합니다.

## Hell-System Keeper 판단 기준

아래 항목에 해당하면 구조 규칙 위반으로 판단합니다.

- 브레이크포인트 값이 공통 토큰이 아닌 컴포넌트 내부에 직접 선언된 경우
- 동일한 브레이크포인트 값이 여러 파일에 반복 선언된 경우

## Hell-Responsive Auditor 판단 기준

반응형 품질 검수 시 아래 항목을 확인합니다.

- 컴포넌트 내부에 임의 숫자 브레이크포인트가 반복 선언되어 있지 않은가
- 브레이크포인트가 공통 토큰 기준을 따르는가
- 레이아웃 전환이 필요한 지점에서만 브레이크포인트가 적용되었는가

## 완료 기준

- 같은 기준값이 여러 파일에 흩어져 있지 않은가
- 반응형 전환 이유를 설명할 수 있는가
- 브레이크포인트 값이 공통 위치에서 관리되는가
