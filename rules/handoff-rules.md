# Handoff Rules

## 목적

- 여러 에이전트가 같은 프로젝트를 이어서 작업할 때 충돌을 줄입니다.

## 규칙

1. 역할 밖 판단은 하지 않습니다.
2. 구조 변경이 필요하면 Hell-Architect로 되돌립니다.
3. 구현 변경은 Hell-Builder가 담당합니다.
4. 검수 단계에서는 새 기능을 임의로 추가하지 않습니다.
5. 콘셉트 단계에서는 구조를 무너뜨리지 않습니다.
6. 인계 시 현재 상태, 변경 파일, 유지 결정, 가정, 다음 작업을 반드시 남깁니다.
7. QA가 같은 문제를 2회 이상 반복 기록했는데 해결되지 않으면 handoff에 `동일 문제 반복으로 인한 재에스컬레이션`을 반드시 명시합니다.
8. 반복 실패 상태의 handoff에는 통과하지 못한 실패 컷 항목 번호를 반드시 함께 적습니다.
9. 구조 변경이 필요한 항목은 Builder 작업으로 넘기지 않고 `Architect 재판단 필요`로 분리해 인계합니다.

## 인계 체크리스트

- 현재 작업 범위가 적혀 있는가
- 변경 파일 목록이 적혀 있는가
- 유지해야 할 결정사항이 적혀 있는가
- 가정으로 남긴 항목이 적혀 있는가
- 다음 담당자 작업이 적혀 있는가
- 반복 실패 여부와 재에스컬레이션 문구가 필요한 경우 포함되어 있는가
- 실패 컷 항목 번호가 필요한 경우 포함되어 있는가
