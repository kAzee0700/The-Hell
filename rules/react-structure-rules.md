# React Structure Rules

## 목적

- React 프로젝트 구조를 일관되게 유지합니다.
- Hell-System Keeper가 구조 규칙 위반 여부를 판단하는 기준을 제공합니다.

## 폴더 책임 원칙

1. `pages`는 라우트 단위 화면만 담당합니다.
2. `components`는 재사용 가능한 UI 조각만 둡니다.
3. `layouts`는 공통 골격과 영역 배치를 담당합니다.
4. `hooks`는 재사용 로직이 명확할 때만 추가합니다.
5. `utils`는 순수 함수, 상수, 설정값 위주로 유지합니다.

## 상태 관리 원칙

1. 전역 상태관리는 실제 복잡도가 확인되기 전까지 도입하지 않습니다.
2. 조기 전역상태 도입은 과도한 추상화로 간주합니다.

## 중복 및 추상화 원칙

1. 같은 책임의 코드가 여러 폴더에 중복되지 않게 유지합니다.
2. 재사용 가능성이 낮은 추상화는 과도하게 추가하지 않습니다.

## 구조 규칙 위반 판단 기준 (Hell-System Keeper)

아래 항목에 해당하면 구조 규칙 위반으로 판단합니다.

- pages에 재사용 가능한 UI 조각이 들어간 경우
- components에 라우트 단위 로직이 들어간 경우
-layouts에 페이지별 로직이 들어간 경우
- 같은 로직이 2개 이상의 폴더에서 중복된 경우
- 전역 상태가 실제 필요 확인 없이 도입된 경우
- 브레이크포인트 값이 공통 토큰 대신 컴포넌트 내부에 직접 선언된 경우

## Builder 운영 기준

- Builder는 구조 규칙을 임의 재해석하지 않습니다.
- 구조 문제가 의심되면 Hell-System Keeper 또는 Hell-Architect 판단을 먼저 받습니다.
- 컴포넌트 내부에 임의 숫자 브레이크포인트를 반복 선언하지 않습니다.

## 완료 기준

- 파일 배치가 책임 기준으로 설명 가능한가
- 재사용 가능성이 낮은 추상화가 과도하게 추가되지 않았는가
- 같은 책임의 코드가 여러 폴더에 중복되지 않았는가
- 브레이크포인트 값이 공통 위치에서 관리되는가
