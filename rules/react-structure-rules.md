# React Structure Rules

## 목적

- React 프로젝트 구조를 일관되게 유지합니다.

## 규칙

1. `pages`는 라우트 단위 화면만 담당합니다.
2. `components`는 재사용 가능한 UI 조각만 둡니다.
3. `layouts`는 공통 골격과 영역 배치를 담당합니다.
4. `hooks`는 재사용 로직이 명확할 때만 추가합니다.
5. `utils`는 순수 함수, 상수, 설정값 위주로 유지합니다.
6. 전역 상태관리는 실제 복잡도가 확인되기 전까지 도입하지 않습니다.
7. 같은 책임의 코드가 여러 폴더에 중복되지 않게 유지합니다.

## 완료 기준

- 파일 배치가 책임 기준으로 설명 가능한가
- 재사용 가능성이 낮은 추상화가 과도하게 추가되지 않았는가
